“Hello World” tutorial - Part II¶

This tutorial expands on the “Hello world!” tutorial and introduces a few new concepts:

  • Comment Blocks
  • Enaml Containers
  • Use of Enaml code from within Python

Enaml supports implementing a model-view-controller architecture , in which it is desirable to separate the data or object model from the view. An ‘.enaml’ file describes the view, and the model is treated by Python code, usually in the form of Traits objects . The Enaml module handles most/all of the controller aspects of the GUI.

Let’s describe a slightly more advanced view in the file (hello_world_view.enaml):

#------------------------------------------------------------------------------
#  This is an Enaml comment block.
#  It looks like a Python comment block.
#------------------------------------------------------------------------------

MyMessageToTheWorld(MainWindow):
    attr my_message
    Container:
        Label:
            text = my_message

Here is the Python code to use the GUI (download here):

#------------------------------------------------------------------------------
#  Copyright (c) 2011, Enthought, Inc.
#  All rights reserved.
#------------------------------------------------------------------------------
import enaml

with enaml.imports():
    from hello_world_view import MyMessageToTheWorld

view = MyMessageToTheWorld(my_message="Hello, world!")
view.show()

Let’s take a closer look at the Enaml file.

The declaration block differs from the previous one in a number of ways.

  • It starts with a comment block.
  • It is not called “Main” since we will call it from a Python program
  • It has an attr, a locally defined attribute
  • The label is put inside a Container, whose primary purpose here is to improve the spacing around the label.

Comment Blocks¶

The .enaml file begins with some comment lines. As in Python code, # begins a comment line and is ignored during lexing and parsing.

#-----------------------------------------------------------------------------
#  This is an Enaml comment block.
#  It looks like a Python comment block.
#-----------------------------------------------------------------------------

Enaml attributes¶

Next, we use a defn block to construct a view that is available for external Python functions to import. The syntax is similar to that of Python compound statements . The header line starts with the Enaml keyword defn and ends with a colon. With the header line, we specify any arguments that will be passed in by the calling Python function. In this case, message is passed in as a parameter.

defn MyMessageToTheWorld(message):
    Window:
        Label:
            text = message

Next, we specify a hierarchy tree of view components. As in Python , indentation is used to specify code block structure. That is, statements beginning at a certain indentation level refer to the header line at the next lower indentation level.

Our view is made up of a Window containing a Label, whose text attribute we set equal to message, which is passed in by the calling function. (We’ll discuss this in more detail in the next tutorial .)

Using the Enaml view in Python¶

Now we’ll take a look at how to use the view in Python code. First, we import Enaml:

import enaml

Then we use enaml.imports() as a context manager for importing the Enaml view.

with enaml.imports():
    from hello_world import MyMessageToTheWorld

Then, we instantiate the view, passing the message to be displayed:

view = MyMessageToTheWorld("Hello, world!")

The show() method on view displays the window:

view.show()
