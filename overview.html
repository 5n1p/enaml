Overview¶

An Enaml application consists of regular Python code, and .enaml files.

A .enaml file is used to describe a GUI as a tree of elements. Each element can associated attributes, and an optional identifier. Attributes customize the layout and behaviour of an application, and identifiers allow Python code to access widgets by name.

Enaml parses a hierarchical file (.enaml), then renders it with an available GUI toolkit. Enaml abstracts away toolkit-specific details.

Goals¶

The goals of the Enaml project are to:

  • Integrate well with Traits and Chaco .
  • Help separate the presentation and content (i.e., MVC)
  • A single script can work across multiple widget toolkits when using the default interfaces.
  • Be extensible and allow adaptation and addition of the base widgets with little effort.

Prerequisites¶

Enaml is developed using Python 2.7 and requires recent versions of the following libraries:

Required:

  • Traits
  • At least one backend toolkit:
    • PySide
    • PyQt4
    • wxPython with a recent (> 0.9.1) agw library
  • PLY (Python Lex-Yacc), for parsing .enaml files
  • cassowarypy Cassowary linear constraint solver.
  • distribute (package installation)

Optional:

  • Graphviz (used for documentation)

Installation¶

To install the package please check out the source from github execute:

python setup.py install

Alternatively you can work in developing mode with:

python setup.py develop
